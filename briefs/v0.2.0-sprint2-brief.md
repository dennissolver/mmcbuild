## Priority 1 — Dashboard onboarding flow

**Problem:** Users can jump directly to Quote or any module without creating a project first, resulting in context-free outputs.

**Required behaviour:**
- On first login OR when no project exists: show a guided onboarding prompt directing the user to create a project before accessing any module.
- On the main dashboard, surface the recommended module sequence clearly as a visual workflow indicator:
  ```
  Build → Comply → Quote → Directory → Training
  ```
- Users must also be able to enter any single module independently (e.g. MMC Comply alone for compliance-only assessment with non-MMC products). The workflow indicator should make both paths explicit.
- "Create Project" must be the clear first step — not buried.

**Implementation notes:**
- If `projects.count === 0` for the user's org: show an onboarding card/banner above the module grid with a prominent "Create Your First Project" CTA.
- The module sequence visual can be a horizontal step indicator or a workflow strip — it does not need to be interactive, just informational.
- Do not gate modules behind a hard redirect if the user actively chooses to use a module standalone. Show a prompt/warning but allow continuation.

## Priority 1 — MMC Build material/system toggle panel

**Problem:** The optimisation module currently makes material/system assumptions automatically without user input before generating outputs.

**Required behaviour:**
- Before analysis runs, the user must see a panel where they can select or deselect individual MMC products and systems. Examples: SIPs, CLT, LSF framing, AAC panels, modular cassettes, 3D printing (flag as indicative), prefab bathroom pods.
- The panel must have a "Save Selection" or "Confirm & Analyse" action.
- The saved material selection becomes the shared design state that flows downstream to:
  1. MMC Build 3D model output
  2. 2D plan export
  3. MMC Quote (cost estimation based on selected materials only)
  4. MMC Comply (compliance check against selected materials only)
- This shared state must persist per project, not per session.

**Implementation notes:**
- Store the material selection against the project record in Supabase (add a `selected_materials` JSONB column to the projects table, or a separate `project_material_selections` table if you anticipate per-analysis versions).
- The material toggle panel should show: material name, category, brief description, an on/off toggle, and optionally a "why this material?" tooltip.
- Default state: all materials toggled ON (opt-out model, not opt-in) so the user sees a full analysis if they just confirm without changes.

## Priority 1 — MMC Direct: admin approval workflow

**Problem:** Directory currently has no submission or curation mechanism. The business model requires a curated, verified directory — not open self-registration.

**Required behaviour:**
1. A public-facing submission form (accessible pre-login, ideally linkable from the Base44 marketing website) where suppliers/professionals can submit their details for consideration.
2. Submitted entries queue for admin review — not published immediately.
3. Admin reviews submission in the admin dashboard and can: Approve (publishes to directory) / Request more info / Reject.
4. Approved entries are published to the live directory.
5. Approved entries are also synced to HubSpot CRM (see Priority 2 below).

**Fields to capture in submission form:**
- Company name, category (dropdown: Builder / Architect / Structural Engineer / Building Certifier / Town Planner / Material Supplier / Trade Contractor / Project Manager / Quantity Surveyor), state(s) of operation, ABN, licence number, contact name, phone, email, website, MMC experience (years), MMC specialisations (multi-select), brief company description.

**Implementation notes:**
- The submission form can live at `/directory/apply` (no auth required).
- Use a `directory_submissions` table in Supabase with a `status` column: `pending | approved | rejected | more_info_requested`.
- Admin review UI goes in the existing Admin Dashboard under "Operations → Directory Admin".
- On approval, copy/promote the submission record into the live `directory_listings` table (or equivalent).
- ABN lookup should run automatically on submission (already built — wire it in).

## Priority 1 — Report/document versioning per project

**Problem:** When users run analysis multiple times (e.g. compliance check at design stage, then again after variation), outputs overwrite rather than accumulating as a version history.

**Required behaviour:**
- Each run of Comply, Build, or Quote generates a new versioned report entry, not an overwrite.
- Users can view the history of reports for a project: "Compliance Check v1 — 28 March 2026", "Compliance Check v2 — 2 April 2026".
- Users can open any previous version.
- The most recent version is shown by default.

**Implementation notes:**
- Add a `report_version` integer column (auto-increment per project per module) to the relevant report/analysis tables.
- The project detail view should have a "Report History" tab or dropdown per module.
- No deletion of previous versions — append only.

## Priority 2 — HubSpot sync for approved directory entries

**Required behaviour:**
- When an entry is approved in the admin workflow (above), push the company record to HubSpot as a new Company object.
- Fields to sync: company name, phone, email, website, ABN, state, category, MMC specialisations.
- Use HubSpot API v3 (Companies endpoint). Store the returned HubSpot `companyId` against the directory record in Supabase for future updates.
- Sync is one-way: Supabase → HubSpot. HubSpot is the CRM of record for sales follow-up; the directory is the source of truth for verified listings.

**Implementation notes:**
- Add `HUBSPOT_API_KEY` to Vercel environment variables (Dennis to add the key).
- Build as an Inngest function triggered on the `directory/entry.approved` event — do not make it a synchronous API call in the approval action.
- Log sync success/failure against the directory record.

**Structural observations to note for future UI work:**
